es6-babel

查看node中已经实现的es6特性

检查运行环境对es6的支持情况

es6转码器Babel的配置
.babelrc是Babel的配置文件，存放于项目的根目录下，用于设置转码规则和插件，基本格式如下
{
“presets”: [ // 设定转码规则
“latest”,
“react”,
“stage-2”
],
“plugins”: []
}

规则集可以根据需要进行安装

不同阶段语法提案的转码规则，stage-0包含范围最广，stage-1次之，依次类推

命令行转码babel-cli
全局安装使用（会导致项目对环境的依赖，无法支持不同项目使用不同版本的Babel）

安装于项目之中

在package.json中添加scripts命令成员
{
// …
“scripts”: {
“build”: “babel src -d lib”
},
}

babel-node是babel-cli自带的命令工具，提供一个支持es6的REPL环境，可以直接运行es6脚本

babel-register改写了require命令，为其加上一个钩子，每当使用require加载后缀为.js,.jsx,.es和.es6的文件时，就会先用Babel进行转码。
require(“babel-register”); //使用时必须先加载
require(“./index.js”);
注：由于是实时转码，所以只适合开发环境

某些代码需要调用Babel的API进行转码，就要使用babel-core

var babel = require(‘babel-core’);

// 字符串转码
babel.transform(‘code();’, options); // => { code, map, ast }
//文件异步转码
babel.transformFile(‘filename.js’, options, function(err, result) {
result; // => { code, map, ast}
});
// 文件同步转码
babel.transformFileSync(‘filename.js’, options); // => { code, map, ast}
// AST转码
babel.transformFromAst(ast, code, options); // { code, map, ast}

options配置内容参考官方文档(babeljs.io/docs/usage/options/)
示例：
var es6Code = ‘let x = n => n + 1’;
var es5Code = require(‘babel-core’).transform(es6Code, {
presets: [‘latest’]
}).code; // ‘“use strict”; \n\nvar x = function x(n) {\n return n + 1; \n};’

Babel默认只转换新的javascript语法，而不转换新的API，如Iterator、Generator、Set、Maps、Proxy、Reflect、Symbol、Promise等全局对象及定义在全局对象上的方法。
如ES6上Array对象新增的Array.from方法，Babel就不会转码。这时想这个方法运行，须使用babel-polyfill为当前环境提供一个垫片。

脚本头部添加如下代码
import ‘babel-polyfill’;
或者
require(‘babel-polyfill’);

可通过查看babel-plugin-transform-runtime模块的definition.js文件查看Babel默认不转码的API。

babel也可用于浏览器环境，Babel6.0版本以后需要使用构建工具构建出来
以Babel配合Browserify为例，首先安装babelify模块

在package.json中添加如下代码，则不必每次都输入参数。
{
“browserify”: {
“transform”: [[“babellify”, { “presets”: [“latest”] }]]
}
}

Babel与ESLint的配合使用

项目根目录下新建.eslintrc,加入parser字段
{
“parser”: “babel-eslint”,
“rules”: {
…
}
}

package.json文件中加入相应scripts脚本
{
// …
“scripts”: {
“lint”: “eslint my-files.js”
},
// …
}

Babel与Mocha的配合使用
package.json中修改scripts.test
{
// …
“scripts”: {
“test”: “mocha –ui qunit” –compilers js:babel-core/register
},
// …
}

–compilers指定脚本的转码器，规定后缀.js的文件都要使用babel-core/register先进行转码
